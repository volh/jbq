#+TITLE: jbq as jq Replacement - Remaining Work
#+DATE: 2026-03-09

* Current State

- [X] Replace the Yojson input path with a native simdjson path.
- [X] Add top-level array streaming for compatible query shapes.
- [X] Preserve huge integers exactly with =BigInt of Z.t=.
- [X] Add exact integer arithmetic for =Int= / =BigInt= values.
- [X] Reach parity with jq on identity-class workloads.
  Current result is near parity, with jq still slightly ahead.
- [X] Beat jq on early-exit and single-pass reduction workloads.
  =where ... | take 5=, =count=, and =map .price | sum= all beat jq on the
  current benchmark data.

* Verified Wins

- [X] Early-exit root-array pipelines are dramatically faster than jq.
- [X] Full-array reductions are materially faster than jq.
- [X] Identity is no longer catastrophically parse-bound as it was with Yojson.
- [X] Big integer literals and arithmetic are better than jq's float-based math.

* Still Open

** Runtime Representation

- [ ] Replace list-backed arrays with a representation optimized for indexing.
  Arrays are still =t list=. This keeps indexing O(n) and hurts locality.

- [ ] Replace list-backed objects with a representation optimized for lookup.
  Objects are still =(string * t) list=. Repeated field access is still O(n).

- [ ] Consider an ordered object representation with a side index.
  Desired properties:
  - preserve insertion order for printing and =keys=
  - fast =.field= lookup
  - avoid unnecessary hashing overhead for tiny objects

** Streaming Coverage

- [ ] Expand the streamed top-array path to cover more query shapes if the
  complexity stays reasonable.

- [ ] Decide whether some fully realizing queries should skip the stream path.
  Identity is near-parity, but not a clear win over jq.

** Build / Portability

- [ ] Improve the simdjson dependency story.

  - use pkg-config
  - document platform requirements more explicitly
