* Setup

- jbq version: 0.1.0 (OCaml 5.4.0, transducer-fused pipelines, direct printer)
- jq version: 1.8.1 (C)
- query-json version: 0.6.1 (OCaml, native binary via npm)
- Benchmarked with [[https://github.com/sharkdp/hyperfine][hyperfine]], warmup 3, min 10 runs
- Platform: CachyOS Linux 6.19.5-3
- Ratio = jbq/other (below 1.0 = jbq wins)

* Analysis

** Where jbq is faster

Small object operations: field access, arithmetic, nested lookups, conditionals.
jbq wins 5/6 vs jq at the single-object scale.

At 1K, transducer-fused operations (where|map, sum, take) are at parity or
faster than jq.

jbq is *2x faster* than query-json on identity and sort_by at all scales.

** Where jq is faster

At scale, jq wins everything by 1.3-1.7x. The gap is consistent — not growing
as fast as before the direct printer optimization.

** jbq vs query-json

jbq dominates query-json at scale:
- identity: 2x faster (772ms vs 1569ms at 100K)
- where: 1.6x faster
- sort_by: 1.7x faster

query-json edges ahead only on sum and count where its lighter parser helps.

** Profiling breakdown (100K, =JBQ_PROFILE=1=)

| Stage            | identity | where  | sum   | where|take 5 |
|------------------+----------+--------+-------+--------------|
| read file        | 11ms     | 11ms   | 11ms  | 11ms         |
| *yojson parse*   | *284ms*  | *288ms* | *290ms* | *279ms*    |
| *of_yojson*      | *96ms*   | *94ms* | *94ms* | *101ms*     |
| parse query      | 0ms      | 0ms    | 0ms   | 0ms          |
| eval pipeline    | 0ms      | 0ms    | 19ms  | 0ms          |
| *output (direct)* | *369ms* | *234ms* | 0ms  | 0ms          |
| print            | 1ms      | 1ms    | 0ms   | 0ms          |

Remaining bottleneck: yojson parse (284ms) + of_yojson (96ms) = *380ms input floor*.
Every query pays this regardless of pipeline complexity.

** Optimization history

| Change              | Identity 100K (jbq/jq) | Notes                         |
|---------------------+------------------------+-------------------------------|
| Seq-based lazy      | 3.10x                 | baseline                      |
| Transducer pipelines | 3.25x                | pipeline ops faster, identity same |
| Direct printer      | *1.51x*               | output: 1098ms → 369ms       |

** Remaining bottleneck

=Yojson.Basic.from_string= (284ms) + =Value.of_yojson= (96ms) = 380ms.
This is the floor for all operations. jq's C parser + NaN-boxed values
handle the same input in ~200ms.

Options to close the remaining gap:
- Direct parser (JSON → Value.t, skip Yojson.Basic.t): saves ~96ms (of_yojson)
- Streaming top-level array into transducer: saves full parse for early-exit ops
