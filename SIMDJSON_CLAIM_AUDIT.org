* Verdicts

- Claim: the branch fully removed Yojson from the repository.
  Verdict: *false*.

- Claim: the experimental CLI input path no longer uses Yojson when
  =JX_EXPERIMENTAL_SIMDJSON_TOP_ARRAY=1= is enabled.
  Verdict: *true*, based on code and profile output.

- Claim: the branch meaningfully improves the targeted workloads over =jbq main=.
  Verdict: *true*.

- Claim: the branch beats =jq= on the tested streamed/reduction workloads.
  Verdict: *true* for the tested commands below.

- Claim: the branch beats =jq= on identity.
  Verdict: *false*.
  It is close, but =jq= is still slightly faster on identity in this audit.

* Final Audit Result

- I was wrong to imply that Yojson was fully removed.
  It was not.

- It *is* correct to say that the experimental CLI input path on this branch,
  with =JX_EXPERIMENTAL_SIMDJSON_TOP_ARRAY=1= enabled, no longer uses Yojson
  for input parsing.

- It *is* correct to say that on the tested commands, this branch beats both
  =jbq main= and =jq= for:
  - =where .price > 400 | take 5=
  - =count=
  - =map .price | sum=

- It is *not* correct to imply a universal memory improvement.
  For the streamed/reduction cases, memory is clearly better than jq in this
  audit. For =identity=, memory is worse than jq.

- It is *not* correct to say it beats =jq= on identity.
  On identity it is roughly at parity, but still slightly slower in this audit.
